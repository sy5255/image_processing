# Phase 8 — 계측 코드용 프롬프트 템플릿

사내 승인 도구(ChatGPT Enterprise)에서 계측 코드를 생성할 때 사용하는 프롬프트 템플릿입니다.
대괄호 `[ ]` 부분을 상황에 맞게 채워 사용하십시오.

> 공통 원칙: **맥락 · 목표 · 입출력 · 제약 · 검증 기준**을 항상 함께 적습니다.
> 받은 코드는 반드시 노트북에서 실행하고, 알려진 시료로 정답과 대조해 검증합니다.

---

## 1. 측정 함수 생성

```
[측정 대상, 예: TEM 흑백 이미지의 밝은 층] 의 [측정량, 예: 두께] 를
재는 파이썬 함수를 작성해줘.

- 입력: [자료형, 예: uint8 grayscale 이미지 img], [scale(nm/pixel)], [기타 인자]
- 방법: [측정 방법을 구체적으로. 예: 세로 라인 프로파일에서 (min+max)/2 를
        지나는 첫/마지막 위치를 선형 보간으로 찾아 두 엣지의 거리(px) × scale]
- 출력: [반환값과 형식. 예: 두께(nm), 실패 시 None]
- 제약: [사용 라이브러리, 예: numpy, cv2 만 사용] / 예외 처리 포함
- 검증: [알려진 정답. 예: 두께 35px, scale 0.25 이면 약 8.75 가 나와야 함]
```

## 2. 다중 위치 반복 측정으로 확장

```
위 [함수 이름] 을 여러 위치에서 반복 측정하도록 확장해줘.
- 입력: 위치 리스트 positions
- 각 위치를 측정하되, 측정 실패(None)는 건너뛰고(continue) 유효값만 모을 것
- 출력: 평균, 표준편차, 측정 개수
- numpy 만 사용
```

## 3. 디버깅 요청

```
아래 코드를 실행하니 [증상, 예: 결과가 정답 8.75 대신 35 로 나옴] 이 발생해.
원인을 찾아 고쳐줘. [의심 지점, 예: scale 환산이 빠진 것 같음]

[코드 붙여넣기]

- 고친 부분과 이유를 함께 설명해줘.
```

## 4. 리팩터링 / 가독성

```
아래 코드를 기능은 그대로 두고 읽기 쉽게 정리해줘.
- 함수로 분리하고 의미 있는 이름 사용
- 각 단계에 짧은 주석
- 동작(결과)은 변경하지 말 것

[코드 붙여넣기]
```

## 5. 검증 코드 요청

```
[함수 이름] 이 올바른지 확인하는 검증 코드를 작성해줘.
- 알려진 정답이 있는 예제 입력 [예: 두께 35px 이미지] 을 만들고
- assert 로 결과가 [기대값 ± 허용오차] 범위인지 확인
```

---

## 검증 체크리스트 (받은 코드마다 확인)

1. 에러 없이 실행되는가 — 존재하지 않는 함수(환각)를 쓰지 않았는가
2. 결과가 상식적 범위인가
3. **알려진 시료로 정답과 대조**했는가
4. 단위 환산(× scale), 색 순서(BGR/RGB), 슬라이싱 경계, 예외 처리가 맞는가
5. 측정 원리가 의도한 방법과 같은가

## 보안 · 지식재산 주의

- 회사가 승인한 도구만 사용합니다.
- 민감한 사내 데이터·이미지·기밀 정보의 입력 여부는 회사 정책에 따라 판단합니다.
- 생성된 코드에도 사내 코드 리뷰·보안 기준을 동일하게 적용합니다.
- 불확실하면 보안·법무 가이드라인을 확인합니다.
